# mybacup

Репозиторий с bash-скриптами для резервного копирования дисков Linux/Windows систем и установки snap-пакетов в Arch Linux.

## Скрипты резервного копирования

### liveBacup.sh
Полная версия для Linux (btrfs + vfat). Создает резервную копию двух разделов:
- `/dev/${disk}1` → vfat (boot)
- `/dev/${disk}2` → btrfs (root)

**Особенности:**
- Интерактивный выбор диска и разделов
- Выбор степени сжатия (gzip -1..-9)
- Создание скрипта восстановления `over.sh`
- Запись метаданных в `readme.txt`

### bacup_win.sh
Полная версия для Windows (ntfs + vfat). Создает резервную копию трех разделов:
- `/dev/${disk}1` → vfat (boot)
- `/dev/${disk}3` → ntfs (system)
- `/dev/${disk}4` → ntfs (data)

### easyBacup.sh
Упрощенная версия для Linux (btrfs + vfat). Автоматически определяет разделы:
- Первый раздел → vfat (boot)
- Второй раздел → btrfs (root)

### easyBacWin.sh
Упрощенная версия для Windows (ntfs + vfat). Автоматически определяет разделы:
- Первый раздел → vfat (boot)
- Второй раздел → ntfs (system)



## Общие требования

### Для скриптов резервного копирования:
- Загрузка с USB установочной флешки Arch Linux
- Права root
- Утилиты: `partclone`, `sfdisk`, `gzip`, `lsblk`
- Раскладка клавиатуры: English (US)

### Структура бекапа:
```
backup_YYYY-MM-DD-HHMM-SS/
├── sda.dump       # Таблица разделов (sfdisk)
├── sda1.pcl.gz    # boot (vfat)
├── sda2.pcl.gz    # root (btrfs/ntfs)
├── readme.txt     # Метаданные
└── over.sh        # Скрипт восстановления
```

## Использование

1. Загрузитесь с установочной флешки Arch Linux
2. Не монтируйте целевой диск к системе
3. Запустите нужный скрипт из текущей директории:
   ```bash
   sudo bash liveBacup.sh
   ```
4. Следуйте интерактивным инструкциям

## Проверка скриптов

```bash
# Проверка синтаксиса
bash -n script.sh

# Проверка всех скриптов
for s in *.sh; do bash -n "$s" && echo "OK: $s" || echo "ERROR: $s"; done

# Зависимости
for cmd in partclone.vfat partclone.btrfs partclone.ntfs sfdisk gzip; do
    command -v "$cmd" >/dev/null 2>&1 || echo "Missing: $cmd"
done
```

## Рекомендации

### Для скриптов резервного копирования:
1. **Перед запуском:** Убедитесь, что целевой диск не смонтирован в систему
2. **Сжатие:** Используйте `-9` для максимального сжатия, если время не критично
3. **Проверка:** После создания бекапа проверьте файлы `readme.txt` и `over.sh`
4. **Хранение:** Храните бекапы на отдельном физическом носителе

### Для archSnap.sh:
1. **Обновление:** Перед установкой рекомендуется обновить систему: `sudo pacman -Syu`
2. **Перезагрузка:** После установки snapd рекомендуется перезагрузить систему
3. **Проверка:** Убедитесь, что сервисы запущены: `systemctl status snapd.socket`

### Общие рекомендации:
1. **Тестирование:** Протестируйте скрипты на виртуальной машине перед использованием на production
2. **Документация:** Внимательно читайте `readme.txt` в каждом бекапе
3. **Восстановление:** Скрипт `over.sh` предназначен для восстановления на тот же диск с той же структурой разделов
4. **Безопасность:** Храните бекапы в защищенном месте, особенно если содержат личные данные
